# 邮件检索系统 — 需求文档

## 1. 项目概述

构建一个基于 AI 的邮件检索问答系统。用户可以通过自然语言提问，系统从历史邮件数据中检索相关信息，返回答案并附上邮件来源。

## 2. 核心功能

### 2.1 邮件数据层

- **假邮件数据生成**：生成一批模拟邮件数据，覆盖多种业务场景
- **邮件字段**：发件人、收件人、抄送、主题、正文、时间、标签、附件列表
- **附件支持**：每封邮件可包含多个附件，附件格式包括：
  - 文档类：`.pdf`、`.docx`、`.xlsx`、`.pptx`、`.txt`、`.csv`
  - 图片类：`.png`、`.jpg`
  - 压缩包：`.zip`
- **附件内容**：附件需有实际可解析的内容（非空文件），以便检索时能覆盖附件内容

### 2.2 检索与问答引擎（后端）

- **邮件索引**：对邮件正文、主题、附件内容建立向量索引（Embedding）
- **语义检索**：根据用户问题进行语义相似度匹配，返回最相关的邮件片段
- **答案生成**：基于检索到的邮件内容，由 LLM 生成回答
- **来源引用**：每个回答必须附带来源信息（邮件主题、发件人、时间、附件名等）
- **上下文感知**：支持多轮对话，后续问题可引用之前的对话上下文

### 2.3 对话管理

- **多对话支持**：用户可创建多个独立对话（类似 ChatGPT 的多会话）
- **对话历史**：每个对话保存完整的消息历史，可随时回顾
- **对话列表**：侧边栏展示所有对话，支持切换、重命名、删除
- **Memory 记录**：系统记住用户的偏好和关注点，跨对话持久化
  - 例如：用户经常查询某个项目的邮件，系统可自动优先检索该项目相关内容

### 2.4 前端界面

- **布局**：左侧对话列表 + 右侧聊天区域（经典 Chat UI）
- **消息展示**：
  - 用户消息：右对齐气泡
  - 系统回答：左对齐气泡，包含答案文本 + 来源引用卡片
- **来源引用卡片**：展示邮件主题、发件人、时间，点击可查看邮件详情
- **附件预览**：来源中涉及附件时，展示附件名和类型图标，支持下载
- **新建对话**：顶部按钮一键新建
- **对话管理**：对话项支持右键菜单（重命名、删除）
- **输入区域**：底部输入框 + 发送按钮，支持 Enter 发送、Shift+Enter 换行
- **加载状态**：检索和生成过程中显示 loading 动画
- **响应式设计**：适配桌面端和移动端

## 3. 技术架构

### 3.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | React + TypeScript + Tailwind CSS | 现代化前端框架，快速开发 |
| 后端 | Python + FastAPI | 高性能异步 API 框架 |
| 向量数据库 | ChromaDB | 轻量级本地向量数据库，零配置 |
| Embedding | OpenAI text-embedding-3-small | 文本向量化 |
| LLM | OpenAI GPT-4o / GPT-3.5-turbo | 答案生成 |
| 数据存储 | SQLite | 对话历史、Memory 等元数据 |
| 文件解析 | python-docx, openpyxl, PyPDF2 等 | 附件内容提取 |

## 4. API 设计

### 4.1 对话管理

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/conversations` | 创建新对话 |
| GET | `/api/conversations` | 获取对话列表 |
| GET | `/api/conversations/{id}` | 获取对话详情（含消息历史） |
| PUT | `/api/conversations/{id}` | 更新对话（重命名） |
| DELETE | `/api/conversations/{id}` | 删除对话 |

### 4.2 聊天问答

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/conversations/{id}/messages` | 发送消息并获取 AI 回答 |

**请求体**：
```json
{
  "content": "Q3 的销售报告发给谁了？"
}
```

**响应体**：
```json
{
  "id": "msg_xxx",
  "role": "assistant",
  "content": "根据邮件记录，Q3 销售报告由张三于 2024-10-15 发送给了李四和王五。",
  "sources": [
    {
      "email_id": "email_001",
      "subject": "Q3 销售报告",
      "from": "zhangsan@company.com",
      "date": "2024-10-15",
      "snippet": "请查收 Q3 季度销售报告...",
      "attachments": ["Q3_销售报告.xlsx"]
    }
  ],
  "created_at": "2025-01-01T12:00:00Z"
}
```

### 4.3 Memory

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/memory` | 获取所有 Memory 条目 |
| DELETE | `/api/memory/{id}` | 删除指定 Memory |

## 5. 假邮件数据规划

生成 **30+ 封邮件**，覆盖以下场景：

| 场景 | 数量 | 附件类型 | 示例 |
|------|------|---------|------|
| 项目进度汇报 | 5封 | `.docx`, `.pptx` | 项目周报、月度汇报 |
| 财务/销售报表 | 5封 | `.xlsx`, `.pdf`, `.csv` | 季度报表、预算表 |
| 会议纪要 | 4封 | `.docx`, `.txt` | 周会纪要、评审会纪要 |
| 合同/法务 | 3封 | `.pdf`, `.docx` | 合同签署、法律审查 |
| 技术文档 | 4封 | `.pdf`, `.txt`, `.zip` | 架构文档、API 文档 |
| 人事行政 | 3封 | `.pdf`, `.xlsx` | 考勤表、通知 |
| 日常沟通 | 4封 | 无/`.png`, `.jpg` | 问题反馈、截图说明 |
| 客户往来 | 4封 | `.pdf`, `.docx`, `.xlsx` | 报价单、需求确认 |

## 6. 非功能性需求

- **响应时间**：问答响应 < 10s（含检索 + LLM 生成）
- **数据安全**：所有数据本地存储，不上传第三方（除 OpenAI API 调用）
- **可扩展性**：索引和检索模块解耦，未来可替换为其他向量数据库或 LLM
- **错误处理**：API 统一错误格式，前端友好提示
